1. Needs & Constraints

Why PostgreSQL + Docker?

  • Reproducibility: Requires identical dev/prod environments (Infrastructure as Code).
  • Relational Needs: SQL is required for the Star Schema design.
  • Cost-efficiency: Open-source tools (PostgreSQL 15, Grafana) minimize licensing overhead.

3. Infrastructure as Code

Containerization strategy

  • docker-compose.yml provisions the database, pgAdmin console, and Grafana.
  • Volumes mapped securely: ./init.sql applies schema on first boot automatically.
  • Virtual Network isolating database from public endpoints.
